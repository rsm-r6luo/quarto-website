Interpretation

Across the linear-model-based methods, impact, trust, and service are the strongest drivers of satisfaction. These three variables rank at the top under Shapley values and Johnson’s relative weights, which are especially useful because they divide shared explanatory power across correlated predictors.

The Pearson correlations are relatively close for several variables, which shows why correlation alone can be misleading in key driver analysis. A variable can look important in a bivariate relationship, but part of that relationship may overlap with other brand perception attributes. The drop-one and standardized regression results help separate unique contribution from shared contribution.

The random forest importances are more evenly distributed than the linear-model-based measures. This suggests that the tree-based model is capturing a somewhat different type of importance, possibly because it can use non-linear splits and interactions. However, the main managerial conclusion is still consistent: improving perceived impact, trust, and service would likely be the most useful focus for improving overall customer satisfaction.

Conclusion

The key driver analysis suggests that customer satisfaction is most strongly associated with whether customers perceive the brand as impactful, trustworthy, and service-oriented. I would prioritize these drivers because they are consistently important across multiple methods, not just under a single metric.

```{python}
def standardize(a):
    """Standardize a vector or matrix using population standard deviation."""
    return (a - a.mean(axis=0)) / a.std(axis=0, ddof=0)


def r2_for_columns(cols):
    """Fit OLS with selected columns and return in-sample R-squared."""
    if len(cols) == 0:
        return 0.0

    X_sub = X[:, cols]
    X_design = np.column_stack([np.ones(len(y)), X_sub])
    beta = np.linalg.lstsq(X_design, y, rcond=None)[0]
    y_hat = X_design @ beta

    ss_res = np.sum((y - y_hat) ** 2)
    ss_tot = np.sum((y - y.mean()) ** 2)
    return 1 - ss_res / ss_tot
```

### Standardized regression coefficients

For standardized regression coefficients, both the outcome and predictors are standardized before fitting the regression. This makes the coefficients comparable across drivers.

```{python}
X_std = standardize(X)
y_std = standardize(y.reshape(-1, 1)).ravel()

X_design_std = np.column_stack([np.ones(len(y_std)), X_std])
standardized_betas = np.linalg.lstsq(X_design_std, y_std, rcond=None)[0][1:]
```

### Drop-one usefulness

Drop-one usefulness compares the full model with a reduced model that removes one driver. A larger value means the model loses more explanatory power when that driver is removed.

```{python}
drop_one = np.array([
    full_r2 - r2_for_columns([j for j in range(len(drivers)) if j != i])
    for i in range(len(drivers))
])
```

### Shapley values / LMG

Shapley values allocate the full model's R-squared across drivers by averaging each driver's marginal contribution across all possible model combinations. This is useful when predictors are correlated because it distributes shared explanatory power more fairly than ordinary regression coefficients.

```{python}
r2_cache = {(): 0.0}


def cached_r2(cols):
    cols = tuple(sorted(cols))
    if cols not in r2_cache:
        r2_cache[cols] = r2_for_columns(list(cols))
    return r2_cache[cols]


p = len(drivers)
shapley_values = np.zeros(p)

for i in range(p):
    other_cols = [j for j in range(p) if j != i]

    for k in range(p):
        weight = math.factorial(k) * math.factorial(p - k - 1) / math.factorial(p)

        for subset in combinations(other_cols, k):
            marginal_r2 = cached_r2(subset + (i,)) - cached_r2(subset)
            shapley_values[i] += weight * marginal_r2
```

The Shapley values sum to **`{python} round(shapley_values.sum(), 3)`**, which matches the full model R-squared.

### Johnson's relative weights

Johnson's relative weights also decompose the model's explained variance across correlated predictors. This approach transforms the original predictors into orthogonal components and then allocates the explained variance back to the original variables.

```{python}
R_xx = np.corrcoef(X_std, rowvar=False)
r_xy = np.corrcoef(X_std, y_std, rowvar=False)[:-1, -1]

eigenvalues, eigenvectors = np.linalg.eigh(R_xx)
eigenvalues = np.maximum(eigenvalues, 1e-12)

R_sqrt = eigenvectors @ np.diag(np.sqrt(eigenvalues)) @ eigenvectors.T
R_inv_sqrt = eigenvectors @ np.diag(1 / np.sqrt(eigenvalues)) @ eigenvectors.T

beta_star = R_inv_sqrt @ r_xy
johnson_weights = (R_sqrt ** 2) @ (beta_star ** 2)
```

The Johnson weights sum to **`{python} round(johnson_weights.sum(), 3)`**, again matching the full model R-squared.

### Random forest mean decrease in Gini

Since `satisfaction` is an ordinal rating, I treat it as a classification outcome for the random forest. The feature importance values below are impurity-based importances, corresponding to mean decrease in Gini for a classification random forest.

```{python}
rf = RandomForestClassifier(

rf.fit(X, y.astype(int))
```
